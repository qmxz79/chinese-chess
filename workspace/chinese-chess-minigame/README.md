# 中国象棋微信小游戏

这是一个中国象棋微信小游戏的开发骨架，当前已具备：规则引擎、基础小程序 UI、联机协议、房间流程与最小信令接入能力。

## 结构

- `src/game/board.js`：棋盘与局面
- `src/game/rules.js`：规则引擎
- `src/game/ai.js`：AI 评估与搜索
- `src/game/match.js`：在线匹配协议抽象
- `miniprogram/`：微信小游戏工程与页面代码
- `scripts/signal-server.cjs`：本地最小联机信令服务
- `test/`：规则、AI 与协议测试

## 运行测试

```bash
npm test
```

## 启动本地联机信令服务

```bash
npm run signal
```

默认监听：

- WebSocket：`ws://127.0.0.1:3200`
- 健康检查：`http://127.0.0.1:3200/healthz`

可通过环境变量覆盖：

```bash
HOST=0.0.0.0 PORT=3200 npm run signal
```

## 最小上机测试路径

1. 启动信令服务：`npm run signal`
2. 在微信开发者工具打开 `miniprogram/`
3. 准备两个客户端（两个模拟器，或一个模拟器 + 一台真机）
4. 两端都填入同一个信令地址
   - 本机双开调试可先试：`ws://<你的局域网 IP>:3200`
   - 注意：微信真机通常不能直连 `ws://127.0.0.1`，需要改成电脑局域网 IP
5. 房主点击“创建房间”
6. 另一端输入房间号后点击“加入房间”
7. 观察是否完成：
   - 双方进入同一房间
   - 红黑方分配明确
   - 双方回合一致
   - 一方落子后另一方能收到同步

## 当前已知限制

- 当前信令服务是“最小可测版本”，主要用于房间/对局链路验证，不是生产版
- 若真机连接失败，优先检查：端口放通、局域网互通、信令地址是否填写为电脑 IP
- 小程序后台若要求 `wss://`，则后续需补 HTTPS/WSS 部署层
